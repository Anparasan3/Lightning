Release

Releasing for web

If you want to run your Lightning app on a device that runs a webbrowser you can call the release-web node script.

Package.json:

{
  "author": "John Smith",
  "name": "Smiths' videos",
  "scripts": {
    "release-web": "node ./node_modules/wpe-lightning-sdk/scripts/release-web.js",
  }
  ...
}

Terminal:

$ npm run release-web

After running the script the dist folder will contain the merged files and is ready to run.

Releasing to the Metrological Dashboard

Releasing to the Metrological appstore requires a few more steps before one can release their app. A token as to be created on http://dashboard.metrological.com in order to run the release script. This token binds the appid to the user who requested the token. Due to this nobody else will be able to overwrite your app.

  1. Make an account
  2. Click on the user icon in the left menu
  3. Select API settings
  4. Select Generate API key
  5. Whitelist all the IP addresses you want to be able to upload from.

After preparing your token, navigate to your project folder with your terminal and run the upload-app node script.

Package.json:

{
  "author": "John Smith",
  "name": "Smiths' videos",
  "scripts": {
    "upload-app": "node ./node_modules/wpe-lightning-sdk/scripts/upload-app-metrological.js",
  }
  ...
}

Terminal:

$ npm run upload-app <your token>

After your app has been uploaded and you are sure your app is production ready please request "App validation" at https://metrological.atlassian.net/servicedesk/customer/portals. By doing this our QA team will start validate your app, if the team spot any bugs they will come back to you in the request ticket, and will ask you to look into these bugs. After the app passes QA the app will be added to our catalog to present to the operators running Lightning.
